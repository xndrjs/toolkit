This site versions its guides by **documentation line**, not by every npm package patch. **v0** is an early, in-motion line: use it to learn the ideas and try the APIs, but expect adjustments as the toolkit settles.

Published packages may already carry their own semver; the **v0** label here means “this narrative and examples target the current pre-stable toolkit shape.” When we cut a stable story, we will add a new docs tree (for example `v1`) and keep **v0** as archive or retire it.
